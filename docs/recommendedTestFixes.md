### New Issue Found (Third Review Pass)

#### [IMPORTANT] `bookEvent()` helper lines 26–28 — sold-out filter is incorrect
`data-testid="book-now-btn"` exists on the `<Link>` element unconditionally — both "Book Now" and "Sold Out" states have it (`EventCard.jsx:111`). If the first matched card is sold out, `#confirm-booking` is disabled and the booking fails.

**Current code:**
```js
const firstCard = page.getByTestId('event-card').filter({
  has: page.getByTestId('book-now-btn'),
}).first();
```

**Fix:**
```js
const firstCard = page.getByTestId('event-card').filter({
  has: page.getByTestId('book-now-btn').filter({ hasText: 'Book Now' }),
}).first();
```

### Issues Found

#### [IMPORTANT] `EventsPage.selectFirstAvailable()` — sold-out filter incorrect
The `data-testid="book-now-btn"` attribute is on the `<Link>` element unconditionally — it exists for both "Book Now" and "Sold Out" states (`EventCard.jsx:111`). The current filter does not exclude sold-out cards.

**Current code (`tests/pages/EventsPage.js`):**
```js
const firstBookable = this.eventCards
  .filter({ has: this.page.getByTestId('book-now-btn') })
  .first();
```

**Fix — also filter by text to exclude sold-out cards:**
```js
const firstBookable = this.eventCards
  .filter({ has: this.page.getByTestId('book-now-btn').filter({ hasText: 'Book Now' }) })
  .first();
```

#### [SUGGESTION] `LoginPage.login()` uses `waitFor` — prefer `expect().toBeVisible()`

**Current code (`tests/pages/LoginPage.js`):**
```js
await this.page.getByRole('link', { name: /Browse Events/i }).first().waitFor({ state: 'visible' });
```

**Fix — produces better failure messages:**
```js
await expect(this.page.getByRole('link', { name: /Browse Events/i }).first()).toBeVisible();
```

## Anti-Pattern Reference

These are the recurring issues found in this codebase — reference before writing new tests:

| Anti-Pattern | Rule | Correct Pattern |
|---|---|---|
| `not.toBeVisible()` to assert element absence | Section 4 | `toHaveCount(0)` |
| `locator('#id')` when `data-testid` also exists | Section 2 (P1 > P4) | `getByTestId('...')` |
| `locator('.css-class')` when `id` or testid exists | Section 2 (P4/P1 > P5) | `getByTestId()` or `locator('#id')` |
| `locator('h3')` plain HTML tag | Section 2 (P5 — never use tag chains) | `getByRole('heading', { level: 3 })` |
| Hardcoding `BASE_URL` as a constant in spec files | Section 1 — Config Reference | Use relative paths; `baseURL` in config |
| Duplicating helper logic inline in a test | Section 5 — Test Structure | Extract to reusable `async function` |
| Step comment numbers not updated after refactor | Section 5 — Comments | Renumber steps after every structural change |
| `filter({ has: getByTestId('book-now-btn') })` to find available events | Section 3 — Filtering | Also filter by text: `getByTestId('book-now-btn').filter({ hasText: 'Book Now' })` |
| `locator.waitFor({ state: 'visible' })` for post-action sync | Section 9 — Wait Strategies | Use `expect(locator).toBeVisible()` — better failure messages |
